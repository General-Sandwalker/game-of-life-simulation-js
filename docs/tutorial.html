Overview

EvoBlob is an evolutionary game theory simulation running entirely in your browser. Circular agents called blobs repeatedly play mathematical games against each other, accumulate scores (energy), reproduce, mutate, and die, creating a living ecosystem that you can observe and tune.

The core game is a variant of the Prisoner's Dilemma: on each round a blob either Cooperates (C) or Defects (D) against its opponent, and payoffs are awarded based on both choices. Over many generations, natural selection favours whichever strategy best accumulates energy in the current population mix.

No server required. EvoBlob is a fully static web page. Open index.html in any modern browser, no installation needed.

Quick Start

  1. Open index.html in your browser.
  2. Review the default parameters in the left panel (or leave them as-is).
  3. Click the green ▶ Start button.
  4. Watch the canvas, blobs will move, interact, reproduce, and die.
  5. Switch chart tabs (Population / Scores / Events) to explore the data.
  6. Hover any blob to see its individual statistics in a tooltip.
  7. Click ⏸ Pause at any time to freeze the simulation, then ⏭ Step to advance one generation at a time.
  8. Click 🔄 New Sim to restart with the current settings, or ↺ Reset to restore all defaults.
Tip: Drag the Speed slider to the right to run hundreds of generations in seconds, then slow down to examine the stable state.

Interface Layout

The page is divided into three columns:

  • Left (Controls), all simulation parameters and action buttons.
  • Centre (Canvas), live animated visualisation of the blob population.
  • Right (Data), real-time charts, strategy rankings, and event log.

A stat bar above the canvas always shows the current generation, total population, cumulative births/deaths/mutations, and the dominant strategy.

Controls Panel

All controls update live, you can adjust parameters while the simulation is running. Most changes take effect on the next generation. The payoff matrix and strategy mix only apply when starting a new simulation.

Simulation Canvas

Each blob is drawn as a soft circle. The colour indicates its strategy. The radius reflects its current score, bigger blobs have more energy. Newborn blobs pulse with a bright glow.

Hover any blob to see a tooltip with its ID, strategy, score, age, win/loss/draw record, cooperation rate, and whether it is a mutant.

Data Panel

Three tabs are available:

  • Population, stacked area chart of how many blobs exist per strategy each generation.
  • Scores, line chart of the average score per strategy over time.
  • Events, stacked bar chart of births, deaths, and mutations per generation.

Below the charts, Strategy Rankings shows a live table sorted by current population, and the Event Log records recent births, deaths, and mutations.

Payoff Matrix

Four values define the Prisoner's Dilemma payoffs. Payoffs are automatically scaled by 1 / (matchesPerGen × roundsPerMatch) to keep scores stable regardless of match frequency.

SymbolNameDefaultWhen earned
TTemptation5You Defect, opponent Cooperates (you win big)
RReward3Both Cooperate (mutual benefit)
PPunishment1Both Defect (mutual loss)
SSucker0You Cooperate, opponent Defects (you lose)
Note: For a valid Prisoner's Dilemma the ordering must be T > R > P > S. Violating this creates different game dynamics (Stag Hunt, Chicken, etc.).

Strategy Mix

Each strategy slider sets its relative starting weight. Weights don't need to add to 100, they are normalised during initialisation. A strategy with weight 0 will not appear at generation 0 (it can still appear later via mutation).

Blob Lifecycle

Birth

Blobs are born either at simulation start (founders) or when a parent reproduces. Reproduction occurs when a blob's score reaches the Reproduction Threshold (default 10). The parent's score is halved and the child receives the other half. The child spawns near the parent and inherits its strategy, unless a mutation occurs.

Playing Matches

Each generation, blobs are randomly paired for N matches. In each match the two blobs play R rounds of the Prisoner's Dilemma, using their memory of past interactions with that specific opponent. Payoffs are immediately added to each blob's score.

Memory

A blob maintains a separate memory for each opponent it has met. This memory holds the last Memory Length rounds, recording both its own move and the opponent's move. Strategies like Tit-for-Tat and Grudger use this to make more informed decisions over repeated interactions.

Aging & Death

At the end of each generation every surviving blob increments its age and pays a Metabolism Cost. Any blob whose score falls below the Death Threshold (default 1.0) or whose age exceeds the Max Age (if set) is removed from the population.

Mutation

When a child is born, there is a Mutation Rate chance it randomly adopts a completely different strategy. Mutations are how rare strategies can re-enter the population and how evolutionary pressure is tested. Mutants are highlighted in the tooltip and log.

Tips & Experiments

Experiment 1, Invasion of the Defectors: Set strategy mix to 100% Always Cooperate, then set mutation rate to 10%. Watch how Always Defect mutants invade and collapse the cooperative population.
Experiment 2, TFT vs Defectors: Start with 50% Tit-for-Tat and 50% Always Defect. TFT should hold its own because defectors punish each other.
Experiment 3, Memory matters: Set Memory Length to 1 vs 20. Short memory makes TFT almost identical to Random in some scenarios.
Experiment 4, Chaos game: Set T=10, S=-5, R=3, P=0 for extreme payoffs. Watch population crashes and explosions.
Performance note: Large populations (>500) with high speed may slow down older computers. Reduce Population Cap if the page feels slow.
